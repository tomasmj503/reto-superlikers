## Verificación del webhook (GET)
Meta golpea la URL con `hub.verify_token` + `hub.challenge`. El nodo compara el token contra `$env.WHATSAPP_VERIFY_TOKEN` y devuelve el challenge en texto plano. La suscripción WABA→app se hizo vía Graph API Explorer con el token permanente de `Admin_Jutilabs`.

## Validación de firma (POST)
HMAC-SHA256 implementado en JS puro en un nodo Code, sin `require` ni `crypto` global. Necesario para ser compatible con el task runner sandboxed de n8n 2.20.

Calcula `hmacSha256Hex($env.WHATSAPP_APP_SECRET, rawBody)` y compara contra el header `X-Hub-Signature-256`. Si no cuadra → `403`. Variables `WHATSAPP_VERIFY_TOKEN` y `WHATSAPP_APP_SECRET` declaradas en `.env` y en `docker-compose.yml` del droplet.
